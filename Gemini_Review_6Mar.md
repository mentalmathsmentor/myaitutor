# Gemini Codebase Review - 6 Mar 2026
**Project:** MAIT (MyAITutor) MVP
**Focus:** Comprehensive Overview of Architecture, Features, and Issues

## 1. Project Organization
The MAIT MVP is a monorepo containing a Python/FastAPI backend and a React/Vite frontend. 
- **Backend:** `backend/` handles student context, fatigue/wellness state, keystroke analytics, bloom's taxonomy pedagogical state, and cloud AI integration (Google Gemini).
- **Frontend:** `frontend/` provides the user interface, incorporating a local "Local Brain" via WebLLM for an in-browser quantized model (Llama-3.2-3B), alongside cloud API fallbacks.

## 2. Core Functional Areas

### 2.1 Backend Architecture
- **Framework:** FastAPI with Uvicorn, structured into routers (in `main.py`) and services (`app/services/`).
- **Student State Management:** Uses a session-based approach with an SQLite database (initialized via `storage.init_db()`), replacing the previous in-memory dict.
- **AI Integration (`educational_agent.py` & `gemini_client.py`):** Uses Google Gemini (`gemini-3-flash-preview`) for complex queries. Prompts are dynamically injected with the student's current fatigue status and Bloom's taxonomy level.
- **RAG / Syllabus:** The primary RAG subsystem using ChromaDB currently has issues (hangs on import) and is bypassed in favor of a FAISS-based retrieval system.

### 2.2 Frontend Architecture
- **Framework:** React + Vite, styled with Tailwind CSS (custom "retro-futuristic cosmic" theme).
- **State Management:** Uses React `useState`/`useEffect`.
- **Local SLM:** The standout feature is the WebLLM integration (`ModelService.js`), allowing the application to run a local LLM in the browser to reduce cloud dependency and latency.
- **Authentication:** Currently a simple placeholder access code (`HSCMATE2026`) in `LandingPage.jsx`, along with Google Single Sign-on integration.

### 2.3 Supporting Pedagogical Systems
1. **Fatigue / Wellness Engine:** Tracks user activity intensity. High intensity reduces AI response complexity or triggers a "Lockout" state to force a break. **Note:** The current formula is fairly aggressive.
2. **Bloom's Taxonomy Engine:** Assesses the cognitive level of the student's query and adjusts teaching strategies accordingly.
3. **Keystroke Psychometrics:** Tracks WPM, dwell time, flight time, and thinking time to categorize student behavior (e.g., "deliberate", "quick", "thoughtful").
4. **Artifact Generation Engine:** Uses LaTeX compilation via Gemini to generate PDF worksheets.

## 3. Critical Findings and Recommendations

### 🔴 High Priority / Security
1. **Remote Code Execution (RCE) Vulnerability:** In `backend/app/services/educational_agent.py:115`, `exec(code_block)` is used to execute Python code generated by the LLM. This is highly dangerous and allows arbitrary code execution on the server if the LLM is prompted maliciously. **Recommendation:** Sandbox this execution environment or remove it entirely.
2. **Hardcoded Secrets:** The frontend access code (`HSCMATE2026`) is hardcoded in the source tree, providing no real security.
3. **CORS Configuration:** `main.py` currently configures CORS to allow specific origins or fall back to localhost, but the configuration logic needs careful review before production deployment to avoid `allow_origins=["*"]` slip-ups.

### 🟡 Medium Priority / Architecture
4. **Fatigue Algorithm Tuning:** The exponential scaling in the wellness engine can lock out students too quickly (e.g., 5 messages in 15 mins). This requires recalibration to prevent user frustration.
5. **RAG Subsystem Stability:** The ChromaDB issues prevent full robust syllabus grounding. Investigate alternative vector stores or fix the Chroma initialization to improve LLM accuracy.
6. **Frontend Bundle Size:** The inclusion of WebLLM and associated dependencies results in a very large bundle (~6.3MB), significantly impacting initial load times. Code splitting strategies should be evaluated.

## 4. Conclusion
MAIT MVP has an incredibly ambitious and forward-thinking feature set, blending psychometrics, adaptive pedagogy, and local/cloud AI architectures. Addressing the `exec()` security vulnerability should be the absolute highest priority before any public external testing. Once stabilized, the unique tutoring adaptations offer a very strong competitive advantage in the EdTech space.
